# サムライ美容室 社内管理システム — 要求定義書

**バージョン**: 1.0
**作成日**: 2026-03-16
**対象システム**: `index.html`（シングルページアプリケーション）

---

## 1. システム概要

サムライ美容室（3店舗）のスタイリストとオーナーが日常業務で使う社内管理ツール。
外部サービス（Square / Googleスプレッドシート / サロンコネクト）と連携し、売上・予約・出勤情報を一元管理する。

### 店舗構成

| 内部キー | Square正式名 | 表示名 |
|---|---|---|
| `nishishinjuku` | 西新宿小滝橋通り | 西新宿本店 |
| `sanchome` | 新宿三丁目 | 新宿三丁目 |
| `shibuya` | 渋谷東 | 渋谷東店 |

---

## 2. ユーザー要件

### 2-1. 利用者

| ロール | 説明 | 権限 |
|---|---|---|
| オーナー | 全店舗のデータを閲覧・管理 | 全機能 |
| スタイリスト | 自分の情報と売上データを確認 | 自分のプロフィール変更・売上閲覧 |

### 2-2. 利用シナリオ

1. **毎日の開店前**: ダッシュボードで本日の予約数・出勤スタッフを確認する
2. **月次または随時**: SquareからCSVをダウンロードし、3店舗の売上を比較する
3. **パスワード忘れ時**: 自分でパスワードを変更する
4. **オーナーが遠隔管理**: どのデバイスからでもブラウザだけで操作できる

| # | 要件 |
|---|---|
| NF-01 | サーバー不要：HTMLファイル1つをブラウザで開くだけで動作する |

## 5. 外部連携

### 5-1. Square（売上CSV）

- Square管理画面からCSVを手動エクスポートし、ドロップアップロードする方式
- リアルタイム連携は対象外（手動運用）
- CSVヘッダーは自動検出（「合計」「売上」「total」「amount」「金額」などを含む列を売上として扱う）

### 5-2. Googleスプレッドシート（ダッシュボード）

- スプレッドシートを「ウェブに公開」し、CSV形式のURLを設定する
- dashboardシートのフォーマットに従いデータを読み込む
- 書き込みは対象外（読み込みのみ）

### 5-3. Co-work（将来実装）

**目的**: サロンコネクトの予約・出勤情報を自動でスプレッドシートに書き込む

**フロー**:
```
サロンコネクト → Co-work（取得・加工） → Googleスプレッドシート → 社内システム（自動読み込み）
```

**Co-workへの指示内容**（設計済み・未実行）:
- サロンコネクトから本日の予約数・出勤人数・スタッフ名を取得する
- dashboardシートの所定セルに書き込む（reservations, staff_count, staff_names）
- 毎日 09:30 / 12:30 / 16:30 に実行し、次の自動読み込みに間に合わせる

## 7. 画面構成

```
ログイン画面
└── メインアプリ
    ├── ヘッダー（ブランド名 / ユーザー名 / ログアウト）
    ├── サイドバー
    │   ├── ダッシュボード
    │   ├── 売上管理
    │   └── プロフィール
    └── コンテンツエリア
        ├── ダッシュボード
        │   ├── 同期ステータスバー
        │   ├── スプレッドシートURL設定
        │   ├── KPIカード（予約数 / 出勤人数 / 次回更新）
        │   └── 出勤スタッフ一覧
        ├── 売上管理
        │   ├── タブ（全店合計 / 西新宿本店 / 新宿三丁目 / 渋谷東店）
        │   ├── 一括アップロードエリア
        │   ├── 店舗別ドロップゾーン（3列）
        │   └── 各タブのデータ表示エリア
        └── プロフィール

            └── パスワード変更フォーム
```
